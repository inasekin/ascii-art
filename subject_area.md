# Описание предметной области

## Тема: Конвертер изображений в ASCII-арт

### 1. Общее описание

Предметная область — система автоматического преобразования растровых
изображений в текстовое ASCII-представление с сохранением истории операций
в реляционной базе данных.

Пользователь загружает изображение (PNG, JPEG, BMP, GIF), выбирает параметры
конвертации и получает ASCII-арт, который можно просмотреть, скопировать или
сохранить в текстовый файл. Каждый факт обработки фиксируется в базе данных.

---

### 2. Сущности предметной области

| Сущность | Описание |
|----------|----------|
| **Изображение (Image)** | Исходный файл, загруженный пользователем. Хранит имя файла, путь, размер в байтах, исходные пиксельные размеры и дату добавления. |
| **Конвертация (Conversion)** | Один акт преобразования изображения. Фиксирует размер ASCII-результата (символов), использованный набор символов, флаги инверсии и цвета, итоговый ASCII-текст и временную метку. |
| **Экспорт (Export)** | Сохранение результата конвертации в файл. Хранит путь назначения, формат (.txt) и дату экспорта. |

---

### 3. Связи между сущностями

- **Изображение → Конвертация**: одно изображение может быть конвертировано
  многократно с разными настройками (связь «один ко многим», 1:N).
- **Конвертация → Экспорт**: результат одной конвертации может быть сохранён
  в несколько файлов (связь «один ко многим», 1:N).

---

### 4. Атрибуты сущностей

#### Изображение (images)
| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | INTEGER PK | Первичный ключ |
| filename | TEXT | Имя файла |
| filepath | TEXT UNIQUE | Полный путь к файлу |
| file_size | INTEGER | Размер файла в байтах |
| width_px | INTEGER | Ширина в пикселях |
| height_px | INTEGER | Высота в пикселях |
| added_at | DATETIME | Дата и время добавления |

#### Конвертация (conversions)
| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | INTEGER PK | Первичный ключ |
| image_id | INTEGER FK | Ссылка на изображение |
| ascii_width | INTEGER | Ширина результата (симв.) |
| ascii_height | INTEGER | Высота результата (симв.) |
| charset | TEXT | Набор символов (standard/extended/gradient) |
| inverted | INTEGER | Флаг инверсии яркости (0/1) |
| colored | INTEGER | Флаг цветного вывода (0/1) |
| result_text | TEXT | ASCII-арт результат |
| converted_at | DATETIME | Дата и время конвертации |

#### Экспорт (exports)
| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | INTEGER PK | Первичный ключ |
| conversion_id | INTEGER FK | Ссылка на конвертацию |
| export_path | TEXT | Путь к сохранённому файлу |
| export_format | TEXT | Формат файла (txt) |
| exported_at | DATETIME | Дата и время экспорта |

---

### 5. Ограничения целостности

- `filepath` в таблице `images` уникален — один файл регистрируется один раз.
- `ON DELETE CASCADE` на `image_id` и `conversion_id` — удаление изображения
  автоматически удаляет связанные конвертации и экспорты.
- `PRAGMA foreign_keys = ON` активируется при каждом соединении с БД.
